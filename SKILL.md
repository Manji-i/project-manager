# 项目管理Skill使用指南

## 核心能力
1. **任务初始化拆解**：拿到项目目标后自动拆解为可落地的子任务，明确每个子任务的Owner、DDL、交付物、验收标准
2. **动态进度维护**：支持实时更新任务进度，自动计算整体完成率，识别延期风险
3. **主动追踪提醒**：按照预设时间节点自动推送任务提醒，同步待办事项
4. **风险识别预警**：自动识别项目卡点、资源缺口、延期风险，给出解决方案建议
5. **定期总结输出**：自动生成周/月报、项目复盘报告，梳理进展、问题、下一步计划
6. **群协作支持**：支持在飞书群内同步项目进展，@对应负责人跟进任务

## 使用流程

### 第一步：项目初始化
当用户提供项目目标、负责人、时间范围等信息时，自动执行：
1. 梳理项目核心目标和衡量标准
2. 拆解为3-8个可落地的子任务，每个子任务明确：
   - 任务名称
   - 负责人（Owner）
   - 预计工时/DDL
   - 交付物要求
   - 依赖关系
3. 生成项目总表，同步给用户确认
4. 自动为每个任务创建飞书日历提醒，配置提前15分钟提醒

### 第二步：日常进度管理
1. **每日跟进**：按照用户指定的时间节点（默认11点/14点/19点）推送当日待办，同步已完成任务进度
2. **进度更新**：用户反馈任务完成/延期时，自动更新项目进度表，重新计算整体完成率
3. **风险预警**：识别到以下情况时自动触发预警：
   - 任务延期超过1天
   - 出现未明确解决的卡点问题
   - 资源缺口未补齐
   预警内容包含：风险描述、影响范围、建议解决方案

### 第三步：定期总结
1. **每周五17:00**：自动生成周进度报告，包含：
   - 本周完成任务清单
   - 整体完成率
   - 待解决问题和风险
   - 下周计划安排
2. **项目结束后**：自动生成项目复盘报告，包含：
   - 目标达成情况
   - 亮点经验
   - 待改进点
   - 后续行动建议

## 配套工具使用说明
### 1. 任务拆解脚本 `scripts/task_split.py`
自动将项目目标拆解为结构化子任务，输入为项目描述、时间范围、负责人列表，输出为JSON格式的任务清单
调用方式：`python scripts/task_split.py "项目描述" "2026-03-01至2026-03-31" "张三,李四,王五"`

### 2. 进度跟踪脚本 `scripts/progress_tracker.py`
维护项目进度表，支持更新任务状态、计算完成率、识别风险
调用方式：
- 更新进度：`python scripts/progress_tracker.py update <项目ID> <任务ID> <完成率> <备注>`
- 查看进度：`python scripts/progress_tracker.py show <项目ID>`
- 风险检查：`python scripts/progress_tracker.py check_risk <项目ID>`

### 3. 报告生成脚本 `scripts/report_generator.py`
自动生成项目周报/复盘报告，输出为Markdown格式
调用方式：`python scripts/report_generator.py <项目ID> <报告类型:weekly/final>`

## 数据存储规范
所有项目数据存储在 `references/projects/` 目录下，每个项目单独一个JSON文件，命名规则：`项目名_创建时间.json`
